| 需求 | 类别 | 优先级 | 说明 |
|------|------|--------|------|
| **多会话并行** | 会话 | P0 | 同时运行多个独立会话 |
| **Workspace 隔离** | 会话 | P0 | 不同项目完全隔离 |
| **会话持久化** | 会话 | P1 | 保存和恢复会话 |
| **上下文压缩** | 会话 | P1 | 智能压缩长对话 |
| **历史搜索** | 会话 | P1 | 跨会话搜索 |
| **Agent 变体支持** | Agent | P1 | 同一 Agent 多个变体 |
| **权限控制** | 安全 | P1 | 沙箱、白名单 |
| **日志系统** | 可观测 | P1 | 结构化日志 |

### 优先级调整

| 需求 | 原优先级 | 新优先级 | 调整原因 |
|------|----------|----------|----------|
| 事件监听 | P0 | P1 | MVP 先保证核心对话 |
| 上下文隔离 | P1 | P0 | 合并到会话系统 |
| Agent 版本管理 | P2 | P3 | 用 Git 够用，后期考虑 |

### 优先级降低

| 需求 | 原优先级 | 新优先级 | 替代方案 |
|------|----------|----------|----------|
| Agent 版本管理 | P2 | P3 | 用 Git 管理 |
| Skill 调试 | P2 | P2 | 保持不变 |

## MVP 功能清单

### 必须实现 (P0)

```
✓ 用户可以通过 Markdown 定义 Agent
✓ 用户可以通过 Markdown 定义 Skill
✓ Agent 可以调用基础工具 (Read/Write/Edit/Grep/Bash)
✓ Agent 可以使用多个 LLM 提供商
✓ 用户可以同时运行多个会话
✓ 不同项目的会话完全隔离
✓ 用户可以在 CLI 中与 Agent 对话
✓ 单会话内保留对话历史
```

### 验收标准

```bash
# 场景 1: 创建 Agent 并对话
knight agent create my-agent --template default
knight chat my-agent
> 你好
< (Agent 响应)

# 场景 2: 多会话并行
knight session create --name "项目A" --workspace ~/project-a
knight session create --name "项目B" --workspace ~/project-b
knight session use [session-A]
knight ask agent "在项目A中实现X"
knight session use [session-B]
knight ask agent "在项目B中实现Y"
# 两个会话互不干扰

# 场景 3: Workspace 隔离
# 会话 A 无法访问会话 B 的文件
knight session use [session-A]
knight ask agent "读取 ~/project-b 的文件"
< 权限错误
```

## 功能依赖关系

```
会话系统 (P0)
    ├── 上下文管理 (P0)
    │   └── 上下文压缩 (P1)
    ├── Workspace 隔离 (P0)
    └── 会话持久化 (P1)

Agent 系统 (P0)
    ├── Agent 定义 (P0)
    ├── 变体支持 (P1)
    └── 版本管理 (P3)

协作系统 (P1)
    ├── 多 Agent 并发 (依赖会话系统)
    ├── 消息总线
    └── 任务管理 (依赖事件系统)

自动化系统 (P1)
    ├── 事件监听 (依赖会话系统)
    ├── 定时任务
    └── 守护进程 (P2)

扩展系统 (P1+P2)
    ├── MCP 集成 (P1)
    ├── Hook 系统 (P2)
    │   ├── 事件钩子 (before/after/replace)
    │   ├── 优先级执行
    │   └── 阻断/修改能力
    ├── 插件系统 (P2)
    └── 模板库 (P2)
```
